**High-Level Architecture**:

- FE → API Gateway/Direct to expressJS (REST + WS).
- Backend services: Auth, Users, Customers, Jobs, Scheduling (calendar logic + optimization stub), Billing, Inventory, Notifications, Reports.
- DB: Normalized with soft deletes, timestamps, companyId scoping, indexes on status/date/techId.
- File storage: Local/uploads → S3.
- Real-time: WS rooms per company/job/tech.
- Scaling: Horizontal pods, DB read replicas, Redis cache, queue workers.

**Key Database Entities** (Prisma schema outline):

- Company (tenant), User (role, permissions JSON or separate), Customer (address, history), Property (multiple per customer), Job (status enum: requested/assigned/enroute/onsite/completed, techId, scheduledAt, duration, priority), JobLineItem/Phase/Checklist, Estimate (linked to Job), Invoice (payments via Stripe), TimeEntry (clock in/out, geofence), InventoryItem (stock, van allocation), Notification, AuditLog.

**Detailed Page-by-Page / Route Specification** (Next.js /app routes, role-gated via middleware/guards):

1. **/login, /register, /forgot-password**: Form, OAuth (Google), magic links. Post-login redirect by role.
2. **/dashboard** (role-specific):
    - Admin/Dispatcher: Widgets - Today's jobs summary, Upcoming/unscheduled count, Revenue pipeline, Technician status (GPS map small), Recommended actions (overdue quotes, past-due invoices), Quick create job/estimate.
    - Technician: My today's jobs list, Quick clock-in, Notifications feed.
3. **/schedule** (main view: FullCalendar month/week/day/agenda + list toggle + map sidebar):
    - Drag-drop assign/reschedule (onEventDrop), resize, color by tech/job-type/status.
    - Filters: Tech, status, priority, date range. Auto-routing stub (nearest + skills). "Dispatch" button → assign + notify.
    - Sidebar: Available techs, unassigned jobs pool. Real-time tech locations (dots on map).
4. **/jobs** (table: DataTable TanStack, filters, pagination, export CSV).
    - Columns: ID, Customer, Address, Status, Tech, Scheduled, Value, Actions.
5. **/jobs/[id]** (tabs: Overview, Checklist/forms (dynamic fields), Photos/videos (upload + before/after), Notes/timeline, Time entries, Expenses/receipts, Change orders, Invoice preview, History/audit).
    - Buttons: Status update (enroute/onsite/complete), "On my way" (SMS ETA), Digital signature (react-signature-canvas), Camera access.
6. **/customers** (list + search) → **/customers/[id]** (profile: contact, properties, service history, jobs/estimates/invoices list, notes, portal link).
7. **/estimates** (list) → create/edit (templates, line items Good/Better/Best, markup calc, PDF preview, digital sign, follow-up automation).
8. **/invoices** (similar to estimates, batch, partial payments, Stripe checkout/link, QuickBooks sync hook).
9. **/inventory** (list, low-stock alerts, purchase orders, assign to job/van, mobile scan stub).
10. **/reports** (dash: Revenue, Profit per job/tech, Utilization, CSAT, Custom builder with filters/charts. AI query stub later).
11. **/settings** (company profile/logo, users/roles/permissions editor, price book/services/materials, business hours, integrations config, notifications templates).
12. Technician mobile-optimized: Collapsed sidebar, large buttons, bottom nav (Schedule, My Jobs, Clock, More).

**API Examples** (expressJS controllers/services):

- GET/POST /jobs, /jobs/:id/status (WS emit), /jobs/assign, /schedule/events (calendar feed).
- Protected with @Roles, @UseGuards(JwtAuthGuard, RolesGuard).
- Webhooks: /stripe, /qb-sync.
